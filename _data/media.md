**云平台性能压测与架构选型**

**项目背景（Situation）**  
公司计划将金融科技从金融系统独立出来，底层需要从硬件采购；基础层面，需要选择合适的私有云虚拟化平台（KVM 或 VMware），需要对两种平台的性能进行全面评估，以指导架构选型。

**项目任务（Task）**  
作为项目支撑人，需设计并执行多维度性能压测（CPU、内存、磁盘、网络、中间件集群），生成对比报告，为公司提供数据化决策依据。

**执行过程（Action）**

- **环境搭建**：
  - 从私有云平台申请 20 台虚拟机，分别在 KVM 和 VMware 上搭建 **LVS + Keepalived + Nginx** 高可用架构、Redis/Zookeeper/RocketMQ 集群，以及基于 Docker 的 Kubernetes 集群。
- **压测设计**：
  
  - 编写测试用例，使用 **Chaosblade、Netperf、WRK、STREAM** 等工具进行压力测试。
    
  - 自定义 Dockerfile 制作 Kubernetes 压测镜像，编写自动化脚本批量执行中间件及集群压测。
    
  - 对两平台虚拟机进行 **2h / 24h / 48h / 1周** 多周期性能测试，涵盖 CPU、内存、磁盘、网络指标。
    
- **数据分析与报告**：
  
  - 收集压测数据与日志，分析性能瓶颈，量化对比两平台在不同场景下的表现。
    
  - 输出性能对比报告及总结，提出针对性优化建议。
    

**成果（Result）**

- 提供覆盖 **计算、存储、网络、中间件** 的全栈性能数据，为公司架构选型提供了核心依据。
  
- 报告被公司采纳，直接影响平台选型决策。
